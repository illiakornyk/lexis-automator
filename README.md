# Lexis Automator

A web-based tool that automates the creation of Anki flashcards for vocabulary learning. The user enters a word, selects the definitions they want, and the system generates a ready-to-import Anki package (CSV + MP3 audio files).

---

## Problem

Creating quality Anki vocabulary cards is a time-consuming, manual process. A learner must:
1. Find a definition from a dictionary.
2. Find or write a usage example.
3. Record or find audio pronunciation.
4. Format everything into Anki's required import format.

This project automates all of those steps into a single web workflow.

---

## Core User Flow (MVP)

1. **Search** — The user types a word (English or German) and the system fetches its definitions and parts of speech from the [Free Dictionary API](https://github.com/meetDeveloper/freeDictionaryAPI) (powered by Wiktionary).
2. **Select** — The user sees a list of definitions grouped by part of speech and selects the ones they want on their card.
3. **Examples** — Each definition shows an example sentence from the dictionary. If no example exists, the user can click a button to generate one via an LLM.
4. **Audio** — The system generates TTS audio for the word and the example sentences.
5. **Web UI & Selection** — The user interacts with a web interface to preview card options and select exactly which definitions/senses should be included in the final export.
6. **Export** — The user downloads a ZIP archive containing a `CSV` file and `MP3` audio files.

## Scope & Non-Goals

To keep the project focused, the following are explicitly out of scope:

- **No mobile app** — desktop web only.
- **No dictionary audio** — all audio is generated by TTS; original dictionary recordings are not fetched.
- **English only** — no RTL or complex tokenization languages.

## Running Locally

### 1. Database Setup (Supabase)
This project uses Supabase for authentication and template cloud sync. You do not need to install the Supabase CLI globally; you can run it via `npx`.

1. Create a Supabase project at [supabase.com](https://supabase.com).
2. Link your local project to your cloud database (you can find your Project ID in your dashboard URL):
   ```bash
   npx supabase link --project-ref <your-project-id>
   ```
3. Push the local database migrations to your cloud project to create the necessary tables:
   ```bash
   npx supabase db push
   ```

### 2. Backend (NestJS + Python Exporter)

```bash
# Navigate to the backend
cd backend

# Install dependencies
pnpm install

# Create a .env file from the template and fill in your Supabase JWT Secret
cp .env.example .env

# Start the development server
pnpm start:dev
```
The API will be available at `http://localhost:3000`. The Swagger UI is at `http://localhost:3000/api`.

### 3. Frontend (Next.js)

```bash
# Navigate to the frontend
cd frontend

# Install dependencies
pnpm install

# Create your .env file
cp .env.example .env.local
# Edit .env.local to include your NEXT_PUBLIC_SUPABASE_URL and NEXT_PUBLIC_SUPABASE_ANON_KEY

# Start the frontend dev server
npm run dev
```
The web interface will be available at `http://localhost:3001` (or your configured port).
